# ディレクトリ構成

Vibrade プロジェクトを効率的に開発・保守するためのディレクトリ構成です。

> 2026-03-16 方針更新:
> 物理演算・衝突判定は pc-client(Phaser) 側で実行し、server はルーム管理と入力中継を担当します。

**モノレポ形式** で複数のパッケージを統一管理します。

# 2. 各ディレクトリの役割

| ディレクトリ | 役割 | 言語 | 説明 |
| --- | --- | --- | --- |
| documents/ | 企画・仕様書 | Markdown | プロジェクトの設計資料 |
| server/ | ゲームサーバ | TypeScript | ルーム管理・接続管理・入力中継を担当 |
| pc-client/ | PC画面 | TypeScript (React + Phaser) | ゲーム表示・QR生成 |
| mobile-client/ | スマホコントローラ | TypeScript (React) | センサー入力・操作送信 |
| shared/ | 共有型・定数 | TypeScript | 3つのパッケージで共有 |
| tests/ | テストコード | TypeScript | ユニット・統合テスト |
| docker/ | デプロイ設定 | Dockerfile | コンテナ化 |

# 3. 責務分離

## server/
**単一責任：接続と入力中継のハブ**

- ルーム作成・参加・切断管理
- 入力イベント受信と room 単位中継
- 参加者リスト配信
- エラー通知とルームのライフサイクル管理

## pc-client/
**単一責任：ゲーム画面表示 + 物理演算**

- Phaser でゲーム描画
- Phaser Scene で物理演算と衝突判定
- QRコード生成＆表示
- プレイヤー一覧表示
- サーバからの入力受信＆ローカル状態更新
- 入力は一切行わない（スマホの役割）

## mobile-client/
**単一責任：操作入力**

- 加速度センサー取得（ジェスチャー検出）
- デバイス傾き取得（ベイ操作）
- 入力をサーバへ送信
- ゲーム画面表示は一切行わない（PCの役割）

## shared/
**単一責任：型安全と定義の統一**

- 全パッケージが参照
- TypeScript の型定義
- 物理定数（MAX_POWER など）

# 4. モノレポ設定（package.json）

ルートの package.json

```json
{
  "private": true,
  "workspaces": [
    "server",
    "pc-client",
    "mobile-client",
    "shared"
  ],
  "scripts": {
    "dev": "yarn workspaces run dev",
    "build": "yarn workspaces run build",
    "test": "yarn workspaces run test"
  }
}
```

### メリット

✅ 統一された依存関係管理  
✅ 共有パッケージの参照が簡単  
✅ 全体ビルド＆テストが一度に実行可能  
✅ monorepo として清潔な構造

# 5. 通信フロー（データ流）

```
スマホ側（mobile-client）
  ↓
  {
    tiltX, tiltY,        # デバイス傾き
    shakePower           # 加速度（発射）
  }
  ↓
Server (game/)
  ↓
  物理計算 → 状態更新
  ↓
  {
    position, velocity,  # 全プレイヤーの位置
    energy, ...
  }
  ↓
PC側（pc-client）
  ↓
  Phaser で描画
```

# 7. 初期開発手順

1. **環境構築**
   ```bash
   yarn install                    # 全 package 依存関係を一括インストール
   yarn dev                        # 全パッケージ dev モード起動
   ```

2. **server/ 優先**
   - ゲームロジック実装
   - Socket.io イベント定義
   - テスト実装

3. **shared/ 整備**
   - サーバが定義した型を shared/ に移動
   - 全クライアントで参照

4. **pc-client/ 実装**
   - Phaser ゲーム画面構築
   - Socket.io リスナー実装

5. **mobile-client/ 実装**
   - センサー取得
   - Socket.io 送信実装

6. **統合テスト**
   - 全パッケージ連動テスト
   - ネットワークテスト

# 8. CI/CD パイプライン

`.github/workflows/` に以下を用意：

- **test.yml**: ユニット＆統合テスト
- **build.yml**: 全パッケージビルド
- **deploy.yml**: Docker ビルド・デプロイ

# 9. トラブルシューティング

| 問題 | 原因 | 解決策 |
| --- | --- | --- |
| shared/ が反映されない | インポートキャッシュ | `yarn cache clean` 実行 |
| 型が同期できない | tsconfig.json の paths 設定 | shared/ を paths に追加 |
| Socket.io イベント名がズレ | 型定義の分散 | shared/types/socket.ts で統一 |
